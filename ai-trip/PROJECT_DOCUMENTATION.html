1. Project Overview

Plan Me a Trip is a full-stack AI-powered travel planning application that helps users create personalized trip itineraries through a conversational AI interface. The application collects user preferences (destination, budget, group size, duration) through an interactive chat and generates a detailed day-by-day itinerary with hotel recommendations, activity suggestions, and cost estimates in Indian Rupees (₹).

Key Value Proposition: Instead of spending hours researching destinations, hotels, and activities, users simply chat with an AI assistant that generates a complete trip plan in under a minute.

Core Workflow

💬 Chat with AI
→
🎯 Pick Preferences
→
✨ AI Generates Plan
→
🗺️ View Itinerary
→
💾 Save & Revisit

6. Backend — Convex Database

6.1 Database Schema

The database uses Convex's schema validation system with two tables:

// convex/schema.ts
defineSchema({
  UserTable: defineTable({
    name: v.string(),        // Display name from Clerk
    email: v.string(),       // Email address
    imageUrl: v.string(),    // Profile picture URL
    subscription: v.optional(v.string()),
  }),

  TripDetailTable: defineTable({
    tripId: v.string(),      // UUID generated on frontend
    tripDetail: v.any(),     // AI-generated JSON trip plan
    uid: v.id("UserTable"),  // Foreign key to UserTable
  }).index("by_uid", ["uid"]), // Index for fast user-specific queries
});

6.2 Mutations (Write Operations)

FunctionFilePurpose
CreateNewUser convex/user.ts Creates a user record (or returns existing) when user signs in via Clerk. Uses email as unique identifier.
CreateTripDetail convex/tripDetail.ts Saves a new trip with tripId (UUID), tripDetail (AI JSON), and uid (user reference).
DeleteTrip convex/tripDetail.ts Deletes a trip by its Convex document _id.

6.3 Queries (Read Operations)

FunctionFilePurpose
GetUserTrips convex/tripDetail.ts Fetches all trips for a user using the by_uid index, ordered newest first.
GetTripByTripId convex/tripDetail.ts Fetches a single trip by its UUID string for the trip detail page.

Why Convex? Unlike traditional REST APIs, Convex provides real-time subscriptions out of the box. When a trip is saved or deleted, all connected browser tabs update automatically — no polling, no WebSocket setup needed. The useQuery hook subscribes to live data.

7. AI Integration — Groq API

7.1 How the AI Works

The application uses Groq's Llama 3.3 70B Versatile model — a powerful open-source LLM served via Groq's ultra-fast inference API. The AI operates in two modes:

Mode 1: Conversational Chat

The AI acts as a trip planner assistant, guiding users through a structured flow:

  1. Understand the origin and destination
  2. Show budget options (Cheap / Moderate / Luxury) via generative UI
  3. Show group size options (Solo / Couple / Family / Friends) via generative UI
  4. Show trip duration selector via generative UI
  5. Confirm all details and show generate button

Each AI response is structured JSON:

{
  "responseText": "Friendly message to the user",
  "ui": "budget"  // Controls which UI component to render
}

Mode 2: Final Trip Generation

When the user clicks "View My Trip", the full conversation is sent with isFinal: true, and the AI generates a complete trip plan:

7.2 Generative UI Pattern

This project implements the Generative UI pattern — the AI doesn't just return text, it controls which interactive UI component appears. The ui field in the AI response maps to React components:

"groupSize"GroupSizeUi4 group type cards (Solo, Couple, Family, Friends)

"none"—Regular text message, no UI component

8. Authentication — Clerk

8.1 Setup

Clerk handles all authentication concerns with zero custom auth code:

  • ClerkProvider wraps the entire app in layout.tsx
  • SignInButton opens Clerk's modal for unauthenticated users
  • UserButton shows the profile avatar with sign-out option
  • useUser hook provides current user data in any component

8.2 User Sync with Convex

When a user signs in via Clerk, the UserDetailProvider (in provider.tsx) calls the CreateNewUser mutation to sync the user into Convex's UserTable. If the user already exists (matched by email), it returns the existing record. This ensures every Clerk user has a corresponding Convex user with a stable _id used for trip ownership.

11. State Management

The application uses React Context API for state management without external libraries like Redux or Zustand:

11.1 UserDetailContext (provider.tsx)

Global context that holds the authenticated user's Convex record. When Clerk auth loads, it calls CreateNewUser mutation and stores the result. Every component can access the user's _id, name, email, and imageUrl.

11.2 TripContext (context/TripContext.tsx)

Scoped to the create-trip page. Enables real-time communication between ChatBox and Itinerary:

  • ChatBox calls setTripData() when the AI generates a trip plan
  • Itinerary reads tripData and renders the plan dynamically
  • This avoids prop drilling and enables the split-screen panels to communicate

11.3 Convex Real-Time Subscriptions

Convex's useQuery hook creates live subscriptions to database queries. On the My Trips page, the trip list auto-updates when trips are added or deleted from any session — no manual refresh needed.

12. API Routes

12.1 POST /api/aimodel

The main AI endpoint. Accepts:

// Request body
{
  "messages": [{ "role": "user", "content": "..." }, ...],
  "isFinal": false  // true for final trip generation
}

// Chat response (isFinal: false)
{
  "responseText": "Where are you starting from?",
  "ui": "none"
}

// Final response (isFinal: true)
{
  "trip_plan": { /* full itinerary JSON */ }
}

Key implementation details:

  • Uses response_format: { type: "json_object" } to guarantee JSON output from the LLM
  • Two different system prompts: CHAT_SYSTEM_PROMPT for conversation and FINAL_SYSTEM_PROMPT for trip generation
  • Graceful error handling with fallback responses

12.2 POST /api/aimodel/google-place-detail

Proxies requests to Google Places API (Text Search) to fetch place photos and details. Uses server-side API key to keep credentials secure.

14. Setup & Running

Prerequisites

  • Node.js 18+ installed
  • npm or yarn package manager
  • Accounts on: Clerk, Convex, Groq Cloud

Installation

# Clone or navigate to project
cd ai-trip

# Install dependencies
npm install

# Start Convex dev server (Terminal 1)
npx convex dev

# Start Next.js dev server (Terminal 2)
npm run dev

# Open in browser
# http://localhost:3000

Production Build

# Build for production
npm run build

# Start production server
npm start
